# 🛡️ SecureBank — AI-Портал Службы Поддержки

**SecureBank** — это веб-система поддержки клиентов для банковской среды. Клиенты отправляют запросы через портал, а встроенный искусственный интеллект (AI) автоматически **классифицирует** их, назначает **приоритет** и предлагает **рекомендуемые действия**. Менеджеры банка обрабатывают эти запросы через удобную панель управления.

UI проекта полностью на **русском языке**.

## 🚀 Подробная инструкция по установке и запуску (Для начинающих)

Если вы запускаете этот проект впервые или не знаете с чего начать, просто следуйте этим шагам:

### Шаг 1: Скачивание необходимых программ
Для того чтобы запустить проект на вашем компьютере, вам потребуется установить платформу Node.js и любой удобный редактор кода.

1. **Установка Node.js** (Обязательно)
   - Перейдите на официальный сайт: [nodejs.org](https://nodejs.org/)
   - Скачайте и установите версию **LTS** (Recommended for most users - Рекомендуется для большинства).
   - *Как проверить, что все установилось?* Откройте терминал (Terminal на Mac или Командную строку на Windows) и введите команду `node -v`. Если программа установлена верно, вы увидите номер версии (например, `v18.17.0` или `v20.x`).

2. **Visual Studio Code** (Рекомендуется)
   - Скачайте удобный редактор кода по ссылке: [code.visualstudio.com](https://code.visualstudio.com/)

### Шаг 2: Получение файлов проекта
Файлы проекта должны находиться у вас на компьютере. Сделать это можно двумя способами:

- **Способ А (Самый простой — Загрузка архива):** 
  На странице проекта в GitHub нажмите на зеленую кнопку **`Code`** и выберите **Download ZIP**. Распакуйте скачанный архив в любое удобное место (например, в папку "Документы" или "Загрузки").
- **Способ Б (Для продвинутых — Через Git):**
  Откройте терминал и выполните команду:
  ```bash
  git clone <ССЫЛКА_НА_ВАШ_РЕПОЗИТОРИЙ>
  ```

### Шаг 3: Открытие проекта и установка зависимостей
Проект состоит из множества библиотек, которые не скачиваются вместе с кодом (чтобы архив был легким). Их нужно скачать командой:

1. Откройте установленный **Visual Studio Code**.
2. В верхнем меню выберите `File -> Open Folder...` (Файл -> Открыть папку...) и выберите разархивированную папку проекта (например, `secure-shield-main`).
3. Откройте встроенный терминал в VS Code. Для этого в верхнем меню нажмите `Terminal -> New Terminal` (или сочетание клавиш `` Ctrl + ` `` на Windows / `` ⌃ + ` `` на Mac).
4. Убедитесь, что вы находитесь внутри папки проекта в терминале. Введите команду для скачивания всех библиотек:
   ```bash
   npm install
   ```
   *Дождитесь окончания загрузки. В проекте появится папка `node_modules` — это значит, что все библиотеки скачаны.*

### Шаг 4: Запуск проекта
После успешной установки библиотек, в том же самом терминале введите команду для старта локального сервера:
```bash
npm run dev
```

В терминале появится текст с зеленой ссылкой (обычно это `http://localhost:5173/` или `http://localhost:8080/`).
- Зажмите кнопку **`Ctrl`** (в Windows) или **`Cmd ⌘`** (на Mac) и кликните по этой ссылке, чтобы проект открылся прямо в вашем браузере.

## ✨ Основной функционал

### 🧑‍💻 Клиентский Портал (`/`)
- Отправка запросов с указанием ФИО, телефона, типа счета и подробным описанием проблемы.
- Мгновенный анализ запроса от AI и выдача уникального номера (ID) обращения.

### 📊 Панель Менеджера (`/dashboard`)
- Просмотр всех запросов в боковой панели, отсортированных по важности (Срочный → Высокий → Средний → Низкий).
- Встроенное отображение аналитики от AI (краткая выжимка проблемы, причины назначения такого приоритета и чек-лист рекомендуемых действий для менеджера).
- Поддержка "Свайпа" (смахивания) для удаления или отметка кейса как **Решено**.

### ⚙️ Настройки AI (`/settings`)
Два независимых режима работы классификатора обращений:
1. **Локальный (по ключевым словам)** — работает оффлайн, встроен прямо в код (ищет совпадения по словам "баг", "украли" и т.д.).
2. **LLM-модель (OpenAI / Anthropic)** — подключение по API-ключу для более глубокого, умного контекстного анализа. 
   *(Ваш API-ключ сохраняется исключительно локально в памяти вашего браузера).*

## 📁 Главные папки проекта (Структура)

```text
src/
├── App.tsx                    # Корневой компонент (маршрутизация и провайдеры)
├── main.tsx                   # Точка входа в React
├── index.css                  # Глобальные стили (в том числе Tailwind базы)
├── pages/                     # Основные страницы сайта
│   ├── CustomerPortal.tsx     # Портал клиента
│   ├── ManagerDashboard.tsx   # Дашборд менеджера
│   └── SettingsPage.tsx       # Страница настроек AI
├── components/                # Переиспользуемые элементы интерфейса (кнопки, карточки)
├── store/                     # Управление глобальным состоянием и кэшем в localStorage
├── hooks/                     # Кастомные хуки (логика AI-классификатора и прочее)
└── types/                     # Типы TypeScript для безопасного кода
```

## 📜 Доступные команды (Scripts)

| Команда | Описание |
|---|---|
| `npm run dev` | Запуск проекта в режиме разработки (с моментальным обновлением при сохранении кода) |
| `npm run build` | Подготовка оптимизированной версии для публикации (продакшена) в папку `dist/` |
| `npm run preview` | Локальный предпросмотр собранной (build) версии проекта |
| `npm run lint` | Запуск проверки кода линтером на ошибки (ESLint) |
| `npm run test` | Запуск автоматических тестов (Vitest) |
